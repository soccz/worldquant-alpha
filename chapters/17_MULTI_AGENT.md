## ICQ3: "왜 실패하는가" — 8대 OS 실패 원인

### 출발점

460개의 알파가 IS(In-Sample)에서는 괜찮아 보이는데
OS(Out-of-Sample)에서 무너지는 패턴을 분석했다.

IS Sharpe 1.5인 알파가 OS에서 0.3이 되는 일이 반복됐다.
OS에서 무너지는 구조적 원인이 있어야 했다.

### 8가지 실패 원인

| # | 실패 원인 | 메커니즘 | 영향 |
|---|---------|---------|------|
| 1 | Crowding | 같은 신호를 쓰는 사람이 많으면 알파가 소멸 | Sharpe 하락 |
| 2 | Regime Shift | 학습 기간과 테스트 기간의 체제가 다름 | 방향 반전 |
| 3 | Lookback Overfit | 특정 윈도우가 IS에서만 우연히 잘 맞음 | OS에서 무의미 |
| 4 | Data Mining | 수백 개를 만들면 우연히 좋은 것이 나옴 | 복수 검정 문제 |
| 5 | Info Speed | 정보가 이미 가격에 반영된 속도보다 늦음 | 알파 = 0 |
| 6 | Universe Drift | TOP3000 구성이 IS와 OS에서 다름 | 종목 불일치 |
| 7 | Linear Assumption | rank() * rank()는 선형 관계만 포착 | 비선형 패턴 놓침 |
| 8 | Lifecycle | 모든 anomaly는 수명이 있다. 발견 → 소멸 | 시간에 따른 약화 |

### ICQ3에서 만든 알파: 35개

8가지 실패 원인을 구조적으로 회피하는 알파 35개를 설계했다.

TIER 1(즉시 제출): 20개
TIER 2(필드 확인 후): 10개
TIER 3(함수 확인 필요): 5개

각 알파에 "어떤 OS 실패를 회피하는가"를 명시적으로 적었다.
Beta-Filtered Revision은 Crowding 회피(저베타 필터로 차별화).
Momentum Crash Alpha는 Regime Shift 직접 포착.
Accrual Anomaly는 30년 검증으로 Lifecycle 위험 낮음.

### 실패: 이론이 성공을 보장하지 않았다

**ICQ3 알파 대부분도 품질 체크에서 탈락했다.**

8대 실패 원인을 "알고 있다"는 것과
"회피하는 알파를 만들 수 있다"는 것은 달랐다.

T1-20(Institutional Accumulation x Revision)은
vwap 필드의 BRAIN 지원 여부를 확인하지 않았다.
T2-1(Alpha Jensen x Revision)은
alpha_jensen 필드의 정확한 이름(접미사)을 몰랐다.

이론은 완벽했다. 실행은 불완전했다.

가장 고통스러운 발견: 35개 중 품질 기준을 통과한 것은
절반도 안 됐을 것으로 추정된다. 정확한 숫자를 확인할 수 없었던 것은
**실제로 전부 시뮬레이션해보지 않았기 때문이다.**

460개에서 배운 "검증 없이 생성만 하는" 습관이
35개에서도 반복됐다.

## ICQ4: "왜 태어날 수 없는가" — BRAIN 제약 자체를 공격

### 전환점: 시스템 파라미터는 중립적이지 않다

ICQ4에서 가장 큰 깨달음은 이것이었다:

> **Neutralization이 알파를 죽인다.**

Subindustry neutralization은 "동일 하위산업 내에서만 비교"를 강제한다.
이것은 산업 간 차이를 포착하는 알파를 원천적으로 차단한다.

예를 들어, "반도체 산업 전체가 소프트웨어 산업보다 저평가"라는 신호는
Subindustry neutralization 하에서 완전히 사라진다.

이건 "알파가 나쁜 것"이 아니라 "시스템이 알파를 죽이는 것"이다.

### "Decay 4가 틀렸다"

기본 Decay 4를 무비판적으로 사용하고 있었다.

하지만 Decay는 알파의 시간 수평선을 결정한다.
분기 업데이트되는 펀더멘탈 신호에 Decay 4를 적용하면
매일 포지션이 불필요하게 변동하고 TVR이 올라간다.

Decay를 8~15로 올리면 TVR이 급감하고 Fitness가 개선된다.

이 단순한 사실을 깨닫는 데 수백 개의 실패가 필요했다.

### ICQ4의 실전 성과

ICQ4는 BRAIN 제약 자체를 공격하는 접근이었지만,
실전에서 만든 알파의 수는 적었다.

"시스템을 이해한다"는 것이 곧 "좋은 알파를 만든다"로
이어지지는 않았다. ICQ4는 진단에 강했지만 처방에 약했다.

## ICQ5: "전제가 맞는가" — 철학적 공격

### 극단적 질문들

ICQ5에서는 알파 생성의 전제 자체를 의심했다:

- **예측이 가능한가?** 효율적 시장 가설이 맞다면 알파는 존재하지 않는다.
- **데이터가 정확한가?** BRAIN의 필드 값이 실제 재무제표와 일치하는지 확인할 방법이 없다.
- **과거 패턴이 미래에 반복되는가?** 구조적 변화(fintech, passive investing)가 anomaly를 영구히 변형할 수 있다.

### 생물학과 열역학에서의 영감

ICQ5에서 가장 야심찬 시도는
생물학과 열역학의 원리를 알파 생성에 적용하는 것이었다.

- **진화론적 알파**: 환경(시장 체제)에 적응하는 알파가 살아남는다
- **엔트로피 알파**: 시장의 무질서도가 증가할 때(변동성 급등) 에너지가 방출된다
- **생태계 알파**: 포식자-피식자 관계처럼 알파는 서로를 소멸시킨다

### 실패: 아름다운 이론, 실전 성과 제로

**ICQ5에서 생물학/열역학 영감으로 만든 알파 중
실전에서 작동한 것은 하나도 없었다.**

이것이 ICQ5의 가장 뼈아픈 교훈이다.

왜 실패했는가:

1. **유비(analogy)는 논리가 아니다.**
   "시장이 생태계와 비슷하다"는 관찰에서
   "생태계의 수식을 시장에 적용한다"로 가는 것은 논리적 비약이다.

2. **추상적 원리는 구체적 수식으로 번역되지 않는다.**
   "엔트로피가 증가한다"를 rank()와 ts_delta()로 어떻게 표현하는가?
   가능한 번역이 수십 가지이고, 어떤 것이 맞는지 알 방법이 없다.

3. **아름다운 이론일수록 과적합 위험이 크다.**
   이론이 복잡하면 데이터에 맞추는 자유도가 커진다.
   그 자유도가 과적합을 만든다.

경제적 이유가 아닌 물리적/생물학적 유비에서 출발한 알파는
전부 실패했다. 예외 없이.

## ICQ6: "실전에서 뭐가 되는가" — 현실 회귀

### 대반전

ICQ3(이론), ICQ4(시스템 공격), ICQ5(철학)를 거치면서
이론의 깊이는 깊어졌지만 실전 성과는 나아지지 않았다.

ICQ6에서 완전히 방향을 바꿨다:

> **확인된 필드만 쓴다. 검증된 패턴만 쓴다. 이론은 최소한만.**

### 확인된 필드 매트릭스

BRAIN에서 실제로 작동하는 필드만 리스트업했다.
미확인 필드는 사용하지 않았다.
ts_zscore, boolean 연산 같은 미확인 함수도 배제했다.

작동이 확실한 것만 남겼을 때,
사용 가능한 빌딩 블록은 의외로 적었다:

- rank(), ts_delta(), ts_mean(), ts_std_dev(), ts_rank()
- 확인된 필드: returns, close, volume, adv20, 기본 fundamental6,
  analyst4 일부, option8 일부, socialmedia12 일부

### 패턴 x 필드 매트릭스

검증된 패턴(반전, 모멘텀, 품질, VRP)과
확인된 필드를 교차하는 매트릭스를 만들었다.

| 패턴 \ 필드 | Fundamental | Analyst | Options | Sentiment | News | PV |
|------------|-------------|---------|---------|-----------|------|-----|
| 반전 | - | - | - | - | - | O |
| 모멘텀 | - | - | - | - | - | O |
| 품질 | O | - | - | - | - | - |
| VRP | - | - | O | - | - | - |
| Revision | - | O | - | - | - | - |
| 교차 | OxO | AxF | OxA | SxA | NxA | - |

### Sales/CF/News 채널: EPS 탈출

가장 큰 실전 발견:
**EPS(earnings_per_share_median_value)에서 벗어나야 한다.**

EPS 기반 알파는 극도로 crowded다.
모든 사람이 EPS revision을 쓴다.
Self-Correlation이 0.7을 넘는 알파의 대부분이 EPS 기반이었다.

대안 채널:
- **Sales**: actual_sales_value_quarterly. EPS보다 조작 어렵다. crowding 낮다.
- **Cash Flow**: cashflow_op, cashflow_invst, cashflow_fin. 3개 CF 교차.
- **News**: news_indx_perf, news_max_up_amt. 완전히 다른 데이터 소스.

```
# Sales 기반 알파 (EPS 대신)
rank(actual_sales_value_quarterly - ts_mean(actual_sales_value_quarterly, 4))
  * (-rank(implied_volatility_call_30 - historical_volatility_30))
```

```
# News 기반 알파
rank(news_indx_perf) * rank(ts_delta(earnings_per_share_median_value, 21))
```

### TIER S: 22개 — 전체 760+에서의 생존자

ICQ1~ICQ6의 전체 760+ 알파에서 최종 선별된 것: **TIER S 22개.**

**생존율: 2.9%.**

460개에서 2%였던 생존율이 760+에서 2.9%로
미세하게 올랐을 뿐이다.

22개의 TIER S 알파에는 공통점이 있었다:

1. **확인된 필드만 사용** — 미확인 필드 하나도 없음
2. **2~3개 데이터셋 교차** — 단일 데이터셋 알파는 하나도 없음
3. **경제적 이유가 한 문장** — "현금흐름 좋은 기업의 과잉 공포 해소"
4. **Crowding이 낮은 조합** — cogs, cashflow_invst, news_indx_perf 같은 미사용 필드

대표적인 TIER S 알파들:

**S1. Accrual Anomaly** — Sloan(1996). 학술적으로 가장 검증된 anomaly.
```
-rank((income - cashflow_op) / (assets + 0.001))
```

**S6. CF Quality x VRP** — 2개 데이터셋 교차.
```
rank(cashflow_op / (assets + 0.001))
  * (-rank(implied_volatility_call_30 - historical_volatility_30))
```

**S14. Triple CF Pattern x Low Vol** — 3개 CF 동시 활용. Crowding 극히 낮음.
```
rank(sign(cashflow_op) + sign(-cashflow_fin) + sign(-cashflow_invst))
  * (1 - rank(historical_volatility_30))
```

**S22. 4중 Consensus** — 4개 완전 독립 데이터셋의 sign() 합. 과적합 위험 최소.
```
rank(
  sign(cashflow_op - income)
  + sign(-ts_delta(sharesout, 60))
  + sign(analyst_revision_rank_derivative - 0.5)
  + sign(-(implied_volatility_call_30 - historical_volatility_30))
)
```

## 감정의 궤적: 이론에 취하고 현실에 깨다

4개 세대를 거치면서 감정의 궤적이 있었다.

### ICQ3: 자신감

8대 실패 원인을 분석했으니 이제 회피할 수 있다.
35개의 알파가 이론적으로 완벽하다.
실행만 하면 된다.

### ICQ4: 분노

시스템이 문제다. Neutralization이 알파를 죽이고,
Decay 기본값이 잘못 설정되어 있다.
시스템을 이해하면 이길 수 있다.

### ICQ5: 도취

생물학과 열역학에서 영감을 받았다.
시장을 완전히 새로운 시각으로 본다.
이 관점이 모든 것을 바꿀 것이다.

→ 실전 성과 제로.

### ICQ6: 체념과 실용

확인된 것만 쓴다. 검증된 것만 쓴다.
이론은 최소한. 데이터가 말하게 한다.
화려하지 않지만 작동한다.

이 궤적을 한마디로: **theory-drunk → reality-check → pragmatic synthesis.**

## 교훈: 세대를 관통하는 패턴

### 교훈 1: 이론의 깊이와 실전 성과는 비례하지 않는다

ICQ5(철학적 공격)가 가장 깊은 이론이었지만 실전 성과는 0이었다.
ICQ6(실용적 접근)가 가장 얕은 이론이었지만 가장 많은 TIER S를 냈다.

### 교훈 2: 확인된 것만 쓰는 것이 가장 강력한 제약이다

미확인 필드를 배제하는 것만으로도 알파의 품질이 올라갔다.
"뭘 쓸 수 있는가"의 제약이 "뭘 만들 수 있는가"를 결정한다.

### 교훈 3: 생존율은 거의 일정하다

460개에서 2%, 760+에서 2.9%.
더 많이 만든다고 생존율이 올라가지 않는다.
더 좋은 것을 만들어야 한다.

### 교훈 4: 데이터셋 교차가 가장 효과적인 차별화다

TIER S 22개 전부가 2개 이상의 데이터셋을 교차했다.
단일 데이터셋 알파는 하나도 살아남지 못했다.

### 교훈 5: 감정의 사이클을 인식하라

자신감 → 분노 → 도취 → 체념.
이 사이클을 인식하는 것이 이성적 판단의 전제 조건이다.
특히 "도취" 단계(ICQ5)에서 가장 큰 시간 낭비가 발생했다.
