# Chapter 01: 알파란 무엇인가

## 우리가 선택한 게임

2026년 3월, 우리는 WorldQuant의 BRAIN 플랫폼 앞에 앉았다. IQC 2026 -- International Quant Championship. 전 세계 대학 팀들이 참가하는 퀀트 알파 생성 대회다. 목표는 단순하다: 주식 시장에서 미래 수익률을 예측하는 수학적 수식, 즉 "알파"를 만들어 제출하는 것.

단순해 보이지만, 이 게임의 규칙을 이해하는 데만 며칠이 걸렸다.

---

## BRAIN이라는 전장

BRAIN은 WorldQuant이 운영하는 클라우드 기반 알파 연구 플랫폼이다. 코드를 짜는 게 아니다. 수식을 입력한다. `rank(ts_mean(returns, 21))` 같은 표현식을 넣으면, 플랫폼이 2010년부터 현재까지의 미국 주식 데이터를 돌려서 백테스트 결과를 보여준다. Sharpe ratio, turnover, fitness, drawdown -- 모든 지표가 자동으로 나온다.

처음에는 이게 쉬워 보였다. 수식 하나 넣고, 결과 보고, 좋으면 제출. 하지만 곧 깨달았다. 이 플랫폼에는 수만 명의 사용자가 이미 수십만 개의 알파를 제출해 놓았다. 우리가 생각하는 "좋은 아이디어"는 이미 누군가 제출한 뒤였다.

Region은 USA만 가능하다. Universe는 TOP3000(미국 유동성 상위 3,000 종목)이 기본이고, TOP200까지 줄일 수 있다. Delay 1이 기본 -- 오늘의 신호로 내일 거래한다는 뜻이다. 이 제약들이 모두 합쳐져서, 같은 공간에서 같은 규칙으로 경쟁하는 구조가 만들어진다.

---

## 점수의 해부학

IQC의 점수 체계를 처음 봤을 때, 가장 먼저 눈에 들어온 건 이 비율이었다.

```python
Merged Performance = 25% * IS Score + 75% * OS Score
```

IS는 In-Sample, OS는 Out-of-Sample이다. 과거 데이터에서 잘 맞추는 것(IS)보다, 처음 보는 데이터에서 작동하는 것(OS)이 세 배 중요하다. 과적합의 유혹을 수학적으로 벌하는 구조다. IS에서 Sharpe 4.0을 찍어도 OS에서 1.0이면 점수가 낮다. 이 사실 하나가 우리의 접근 방식을 근본적으로 바꿔놨다.

IS Score도 단순하지 않다. `D1 Score + D0 Score / 3`으로 구성되어 있어서, Delay 1 알파가 메인이고 Delay 0은 보너스 정도다. 그리고 하루에 쌓을 수 있는 점수는 최대 2,000점으로 제한된다. 아무리 좋은 알파를 많이 만들어도, 하루에 2,000점이 한도다. Gold 레벨(10,000점)에 도달하려면 최소 5일은 매일 꾸준히 제출해야 한다.

이건 스프린트가 아니라 마라톤이다.

## 데이터라는 무기고

BRAIN에서 사용할 수 있는 데이터셋은 14개, 필드는 총 2,645개다. 하지만 모든 데이터셋이 동등하지 않다.

| 데이터셋 | 필드 수 | Coverage | 경쟁 강도 |
|----------|---------|----------|-----------|
| Price Volume | 23 | 100% | 극심 |
| Analyst Estimate | 653 | 72% | 중간 |
| Company Fundamental | 886 | 50% | 높음 |
| Options Analytics | 74 | 70% | 낮음 |
| US News | 322 | 80% | 중간 |
| Relationship Data | 165 | 80% | 낮음 |
| Report Footnotes | 318 | 41% | 매우 낮음 |
| Sentiment | 18 | 99% | 높음 |
| Volatility Data | 64 | 69% | 중간 |

Price Volume Data는 coverage 100%라 모든 종목에 데이터가 있다. open, close, high, low, volume, returns -- 가장 기본적인 필드들. 그리고 바로 그것이 문제다. 모든 사람이 접근할 수 있는 데이터로 만든 알파는 이미 crowded 되어 있다.

반면 Analyst Estimate는 653개 필드에 72% coverage다. Options Analytics는 74개 필드에 70% coverage. Report Footnotes는 318개 필드에 41% coverage밖에 안 된다. Coverage가 낮다는 건 데이터가 없는 종목이 많다는 뜻이지만, 동시에 그 데이터를 사용하는 경쟁자도 적다는 뜻이다.

나중에 알게 된 사실이지만, 이 데이터셋 선택이 알파의 생사를 결정했다.

## 좋은 알파의 조건

BRAIN이 알파를 받아주려면 네 가지 관문을 통과해야 한다.

```python
# 제출 최소 기준 (Delay 1)
Sharpe >= 1.25          # 위험 대비 수익
Fitness >= 1.0          # 종합 성과 지표 (Average 이상)
1% < Turnover < 70%    # 너무 자주 또는 너무 안 바꾸면 탈락
Self_Correlation < 0.7  # 기존 제출 알파와 충분히 달라야 함
```

Sharpe ratio는 수익률을 변동성으로 나눈 값이다. 1.25 이상이어야 한다. 이게 쉬워 보이지만, OS 기준으로 1.25를 넘기는 건 완전히 다른 이야기다.

Turnover는 포트폴리오가 매일 얼마나 바뀌는지의 비율이다. 1% 미만이면 너무 정적이고, 70% 이상이면 거래비용이 수익을 잡아먹는다. 대부분의 초보 알파는 TVR이 너무 높아서 탈락한다.

Self-Correlation은 이미 제출한 알파들과의 상관관계다. 0.7 이상이면 "이미 비슷한 거 냈잖아"라는 판정을 받는다. window 크기만 살짝 바꿔서 여러 개 내는 전략은 통하지 않는다.

그리고 가장 중요한 Fitness. 이 공식을 처음 봤을 때 한참을 들여다봤다.

```python
Fitness = Sharpe * sqrt(abs(Returns) / max(Turnover, 0.125))
```

Fitness는 Sharpe에 효율성 계수를 곱한 값이다. Returns(수익률)을 Turnover(회전율)로 나눈 비율의 제곱근 -- 즉, 같은 Sharpe라도 적게 거래하면서 달성한 쪽이 Fitness가 높다. 분모의 `max(TVR, 0.125)`는 turnover가 12.5% 이하로 내려가도 추가 이득이 없다는 뜻이다.

이 공식이 의미하는 바는 명확하다. BRAIN은 **효율적인 알파**를 원한다. 많이 거래하면서 Sharpe를 높이는 건 가치가 없다. 적게 거래하면서도 안정적인 수익을 내는 수식이 진짜 알파다.

## 게임의 본질

모든 규칙을 이해하고 나서, 이 게임의 본질이 보이기 시작했다.

이건 단순히 "좋은 수식을 찾는" 문제가 아니다. 수만 명이 같은 데이터를 보고, 같은 플랫폼에서, 같은 기준으로 경쟁한다. 교과서에 나오는 팩터는 이미 다 제출되어 있다. 모멘텀, 가치, 저변동성 -- 전부 crowded다.

그렇다면 어디서 edge를 찾을 것인가?

후에 알게 되지만, 답은 "남들이 보지 않는 데이터셋의 교차점"에 있었다. 하지만 그 답에 도달하기까지, 우리는 먼저 벽에 부딪혀야 했다. 460개의 알파를 만들고, 대부분이 실패하는 경험을 해야 했다.

이것은 우리가 선택한 게임이다. 규칙은 정해져 있고, 데이터는 공유되며, 기준은 냉정하다. 이제 시작이다.
