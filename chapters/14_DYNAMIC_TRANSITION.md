# 14장: 동적 전환 — 비판에서 태어난 새 방법론들

---

## 서문: 정적에서 동적으로

13장의 자기 비판에서 한 가지 핵심이 드러났다:

> 지금까지 만든 것들의 가장 큰 문제는 **정적(static)**이라는 것이다.

"지금 이 신호가 높다" → 매수/매도.

하지만 시장은 동적(dynamic)이다.

- 신호가 임계값을 돌파하는 순간
- 신호의 패턴이 바뀌는 순간
- 여러 신호가 동시에 같은 방향으로 수렴하는 순간
- 두 신호가 서로 불일치하는 순간

이 전환점과 불일치가 진짜 알파다.

12_information_cascade_speed.md가 460개 중에서 수준이 달랐던 이유도 이것이다.
단순히 "애널리스트 수정이 높다"가 아니라
"애널리스트 수정이 비정상적으로 높은데 가격이 아직 안 움직였다"는
두 신호의 불일치와 전환점을 포착했기 때문이다.

이 장에서는 비판에서 태어난 5가지 새 방법론과,
사이클 위치 감지, 그리고 실패 패턴 역이용 이론을 다룬다.

## 방법론 A: 임계값 돌파 구조

### 아이디어

정보 전파는 선형이 아니다. 작은 신호는 무시되고,
임계값을 넘는 순간 갑자기 전파가 시작된다.

기관 투자자의 행동 패턴:
- 애널리스트 수정 1번: 노이즈로 처리
- 3번 연속: 포지션 검토 시작
- 5번 연속 + 크기 증가: 실제 매수 실행

이 임계값 돌파 직후가 가장 강한 알파 구간이다.

### 구현

```
# 신호가 역사적 상위 25%에 최근 23일 내 처음 진입한 순간
rank(
  ts_rank(earnings_per_share_median_value, 233)
  - ts_lag(ts_rank(earnings_per_share_median_value, 233), 23)
)
  * (ts_rank(earnings_per_share_median_value, 233) > 0.75)
```

경제적 논리: EPS 추정치가 자기 역사의 상위 25%에 최근 진입.
이미 오래 상위에 있던 종목은 이미 반영됨 → 자동 제외.
돌파 직후 23일이 가장 강한 알파 구간.

### 실패와 한계

이론적으로 아름다웠지만 실전에서 문제가 있었다:

**실패 1: (> 0.75) boolean 연산의 BRAIN 지원 미확인.**
BRAIN 파서가 비교 연산자를 어떻게 처리하는지 검증하지 않았다.
이것 하나로 방법론 A의 절반이 무효화될 수 있었다.

**실패 2: ts_rank의 불연속성.**
ts_rank는 이산적이다. 233일 윈도우에서 랭크가 0.75를 넘는 순간이
정확히 "돌파"를 의미하는지, 아니면 데이터 해상도의 아티팩트인지 구분이 안 된다.

그럼에도 이 방법론은 "정적 수준"이 아니라 "동적 변화"를 포착한다는 점에서
이전과 질적으로 달랐다.

## 방법론 B: 시계열 패턴 알파

### 아이디어

신호의 현재 수준이 아니라, 신호의 역사적 패턴이 정보다.

EPS revision이 3분기 연속 상향이면 4분기도 상향일 확률이 높다.
이건 현재 revision 수준이 아니라 revision의 시계열 패턴이다.

### 구현

```
# 연속 상향 수정 (revision streak)
# 최근 89일간 양수 수정 횟수를 세고, 3회 이상일 때만 활성화
revision_positive = ts_delta(earnings_per_share_median_value, 23) > 0

rank(ts_sum(revision_positive, 89))
  * (ts_sum(revision_positive, 89) > 2)
```

또 다른 패턴 — 고변동성 → 저변동성 전환:

```
# 과거 89일 중 20일 이상 고변동성이었고 지금은 조용해진 상태
was_volatile = ts_rank(historical_volatility_20, 233) > 0.7
is_quiet = ts_rank(historical_volatility_20, 233) < 0.4

rank(ts_mean(returns, 63))
  * (ts_sum(was_volatile, 89) > 20)
  * is_quiet
```

### 실패

**이 방법론은 BRAIN에서 가장 많이 실패했다.**

이유: boolean 변수의 ts_sum() 지원 미확인.
`ts_delta(x, 23) > 0`이 true/false를 반환하는지,
그걸 ts_sum()으로 합산할 수 있는지 전혀 검증하지 않았다.

경제적 논리는 문헌에서 검증된 것이었다 —
Jegadeesh & Titman(1993)의 모멘텀을 revision에 적용한 것.
하지만 플랫폼 제약을 무시한 채 이론만 쌓았다.

교훈: **아무리 좋은 이론이라도 실행 환경에서 구현 불가능하면 무가치하다.**

## 방법론 C: 신호 간 불일치 알파

### 아이디어

두 신호가 반대 방향을 가리킬 때, 어느 쪽이 맞는지가 알파다.

### 핵심 구현: "기관이 속고 있다"

```
# 미시구조 좋음(기관이 매수 중) + 이익 품질 나쁨 = 기관이 속는 중
micro_signal = ts_zscore((close - vwap) / (close + 0.001), 47)
quality_signal = ts_zscore(cashflow_op / (abs(income) + 0.001), 131)

# 미시구조 좋은데 품질 나쁨 → 숏
-rank(micro_signal)
  * (micro_signal > 0.5)
  * (quality_signal < -0.5)
```

기관은 미시구조(VWAP, 거래량)를 보고 매수 중이다.
하지만 이익 품질은 악화 중. 결국 실망 매도가 온다.
두 신호의 불일치 = 정보 비대칭의 극단.

### 다른 불일치 패턴

**애널리스트 낙관 + 옵션 시장 비관:**

```
# 애널리스트는 상향 수정 중, 옵션 시장(IV)은 상승
# 옵션 시장은 informed trader가 선호 → 옵션 시장이 맞을 가능성
-rank(ts_delta(earnings_per_share_median_value, 23))
  * (ts_delta(earnings_per_share_median_value, 23) > 0)
  * (ts_delta(implied_volatility_call_30, 13) > 0)
```

**센티먼트 과열 + 미시구조 약화:**

```
# 센티먼트 높음 + VWAP 괴리 낮음(기관 매수 없음) = 소매만 들어온 것
-rank(ts_zscore(scl12_sentiment, 47))
  * (ts_zscore(scl12_sentiment, 47) > 0.7)
  * (ts_zscore((close - vwap) / (close + 0.001), 47) < 0)
```

### 실패

**3개 곱의 신호 희소 문제.**

micro_signal > 0.5이면서 quality_signal < -0.5인 종목이
TOP3000에서 몇 개나 되는지 확인하지 않았다.

이론적으로는 "극단적 불일치"가 강한 신호지만,
해당 종목이 50개 미만이면 포트폴리오가 너무 집중된다.
과적합 위험이 급증하고 TVR이 불안정해진다.

**이것이 "사고는 4중, 수식은 2~3중"이라는 원칙의 탄생 배경이다.**

## 방법론 D: 위치 x 속도

### 아이디어

지금 어디에 있는지(위치) x 얼마나 빠르게 변하는지(속도).
위치만 보거나 속도만 보는 것보다 강하다.

### 핵심 구현: "역사적 저점 + 빠른 개선 = 최적 진입"

```
position = ts_rank(cashflow_op / (assets + 0.001), 233)
velocity = ts_delta(cashflow_op / (assets + 0.001), 47)

rank(velocity)
  * (position < 0.3)   # 역사적 하위 30%에 있고
  * (velocity > 0)      # 지금 개선 중
```

역사적 저점에서 개선이 시작되는 종목이 최대 회복 잠재력을 가진다.
이미 역사적 고점에서 개선 중인 것보다 훨씬 강한 신호.

### 실패

**분기 데이터의 velocity 문제.**

cashflow_op는 분기 업데이트다. ts_delta(47)이면
분기 발표 직후 한 번만 non-zero이고 나머지 46일은 0이다.

velocity가 0이면 position < 0.3인지 아닌지가 의미 없다.
대부분의 날짜에서 이 알파는 아무 것도 하지 않는다.

일일 업데이트되는 필드(returns, historical_volatility)로
velocity를 측정해야 했다. 분기 데이터는 position에만 써야 했다.

이 실수를 깨달은 건 25_PLAN_AND_VERIFY의 Verifier 분석에서였다.

## 방법론 E: 정보 밀도 알파

### 아이디어

같은 기간에 얼마나 많은 독립적 정보가 들어왔는가.
정보 밀도가 높은 구간 직후가 가장 강한 알파 기회다.

### 구현

```
# 3개 독립 소스에서 동시에 비정상 변화 발생
info_density =
  (abs(ts_delta(earnings_per_share_median_value, 23)) > 0)
  + (abs(ts_delta(implied_volatility_call_30, 13))
     > ts_std_dev(ts_delta(implied_volatility_call_30, 13), 89))
  + (abs(ts_delta(scl12_sentiment, 11))
     > ts_std_dev(ts_delta(scl12_sentiment, 11), 67))

# 정보 밀도 높음 + 가격 미반영 = 최대 알파 기회
rank(info_density) * (rank(ts_mean(returns, 5)) < 0.5)
```

여러 소스에서 동시에 정보가 들어오면 = 진짜 중요한 사건.
하나의 소스만 변하면 노이즈, 3개 동시 변화는 신호.
가격이 아직 안 움직였다면 = 정보 처리 지연 극대화.

### 실패

**이 방법론은 BRAIN 문법에서 가장 불확실했다.**

boolean 합산, abs(), 비교 연산자(>) — 모두 미확인.
경제적 논리는 가장 설득력 있었지만, 플랫폼 구현 가능성은 가장 낮았다.

19_CRITIQUE_AND_NEW_METHODS의 결론에서도 인정했다:
"방법론 B와 E는 BRAIN 문법 확인 필요."

아름다운 이론이 실행 불가능한 코드로 끝나는 경험.
이것이 460개의 주된 실패 패턴이었다.

## 사이클 위치 감지: 4-state 사이클

5가지 방법론을 넘어서, 20_cycle_position_detection에서
더 근본적인 프레임워크를 개발했다.

### 4-State 사이클

시장은 Bull/Bear x quiet/volatile의 4-state 사이클을 반복한다.
각 상태에서 작동하는 anomaly가 다르다:

| 상태 | 모멘텀 | 반전 | 가치 |
|------|--------|------|------|
| Bull/quiet | 유지 | 약함 | 약함 |
| Bull/volatile | 약함 | 중간 | 약함 |
| Bear/quiet | **최강** | 약함 | 유지 |
| Bear/volatile | **크래시** | **최강** | 유지 |

같은 모멘텀 신호도 사이클 위치에 따라 IC가 5배 차이가 났다.
Bear/quiet에서 모멘텀 프리미엄이 극대화되고,
Bear/volatile에서 모멘텀이 크래시한다.

### BRAIN에서의 구현

문제: BRAIN은 개별 종목 수준이다. "시장 사이클"은 시장 전체 개념이다.
해법: 종목 자체의 시계열 정보로 사이클 위치를 근사한다.

```
# 변동성 위상: 단기/장기 vol 비율
rank(ts_std_dev(returns, 11) / ts_std_dev(returns, 61))

# < 1이면 quiet phase, > 1이면 volatile phase
```

```
# 가격 위치: 1년 내 현재 위치
ts_rank(close, 251)

# 0.0~0.25 = Bear 바닥, 0.75~1.0 = Bull 과열
```

### 조건부 전략 전환

```
# quiet phase에서만 모멘텀 활성화
rank(ts_mean(returns, 23))
  * (bucket(rank(ts_std_dev(returns, 11) / ts_std_dev(returns, 61)), 3) == 1)
```

변동성 비율을 3분위로 나누고, 하위 1/3(quiet)에서만 모멘텀 사용.
volatile phase에서 모멘텀이 꺼지므로 momentum crash 회피.

### 실패: bucket() 경계 불안정

**bucket() 함수의 경계에서 종목이 빈번하게 이동한다.**

변동성 비율이 경계 근처에 있는 종목은
하루는 quiet, 다음 날은 volatile로 분류된다.
이 종목들의 포지션이 매일 바뀌면서 TVR이 폭발한다.

bucket()의 이산적 분류가 연속적 신호에 적합하지 않았다.
결국 곱셈 필터(rank(signal) * rank(state))가 더 안정적이었다.

## 실패 패턴 역이용: 역발상의 알파

37_FAILURE_MODE_EXPLOITATION에서 완전히 다른 관점을 시도했다.

### 핵심 질문

> 알려진 알파가 실패하는 조건을 알면, 그 실패 자체를 알파로 쓸 수 있는가?

모든 알파는 특정 조건에서 실패한다.
알파가 실패하는 순간:
1. 그 알파를 따르던 투자자들이 손실을 보고 청산한다
2. 청산이 가격을 반대 방향으로 밀어낸다
3. 이 과정이 예측 가능하다 (실패 조건이 학술 문헌에 잘 정리되어 있으므로)

실패 조건 + 실패 방향 = 반대 방향 알파.

### 모멘텀 크래시를 거래 기회로

Daniel & Moskowitz(2016)이 밝힌 모멘텀 크래시 조건:
Bear market + 고변동성.

이 조건을 **모멘텀 반대 방향 알파**로 직접 구현:

```
# FM-1a: 모멘텀 x volatile 체제 = 모멘텀 크래시 포착
-rank(ts_mean(returns, 63))
  * rank(historical_volatility_10 / (historical_volatility_60 + 0.001))
```

모멘텀이 강한데 변동성이 급등하는 종목 → 모멘텀 반전 직전.
모멘텀 추종자들이 아직 포지션을 유지 중 → 청산하면 급락.

### 가치 함정을 거래 기회로

```
# FM-3: 저PBR + 이익 품질 악화 = 가치 함정
-rank(bookvalue_ps / (close + 0.001))
  * (1 - rank(ts_delta(cashflow_op / (assets + 0.001), 252)))
```

가치 투자자들이 저PBR을 보고 매수한다.
하지만 현금흐름이 악화 중이면 장부가가 곧 하락한다.
가치 투자자들이 손실을 보고 청산할 때 추가 하락.

### 센티먼트 버블을 거래 기회로

```
# FM-8: 센티먼트 높음 + 거래량 급증 = 소매 과열
-rank(scl12_sentiment)
  * rank(ts_zscore(volume / (adv20 + 0.001), 47))
```

개인 투자자 과열 후 청산 → 급락.
센티먼트가 높을수록, 거래량이 비정상적일수록 강한 숏.

### 실패 패턴 이론의 한계

**이론은 매력적이었지만, 실전 결과는 혼재했다.**

문제 1: 실패 조건이 "알려져 있다"는 것은 양날의 검이다.
학술적으로 알려진 실패 패턴은 이미 다른 사람들도 거래하고 있을 수 있다.
모멘텀 크래시 방어 전략은 이미 crowded일 가능성.

문제 2: 실패 조건의 타이밍이 불확실하다.
모멘텀이 크래시할 "것"이라는 건 알지만, "언제" 크래시할지 모른다.
Decay 4로 매일 숏을 잡으면 크래시 전까지 손실이 누적된다.

## 5가지 방법론의 종합 평가

| 방법론 | 이론적 강도 | 실행 가능성 | 실패 원인 |
|--------|-----------|-----------|----------|
| A. 임계값 돌파 | 높음 | 중간 | boolean 연산 미확인 |
| B. 시계열 패턴 | 높음 | 낮음 | boolean ts_sum 미확인 |
| C. 신호 불일치 | 매우 높음 | 중간 | 3중 곱 신호 희소 |
| D. 위치 x 속도 | 중간 | 중간 | 분기 데이터 velocity = 0 |
| E. 정보 밀도 | 매우 높음 | 매우 낮음 | 거의 모든 문법 미확인 |

460개의 실패에서 태어났지만, 이 5가지도 또 실패의 씨앗을 품고 있었다.

하지만 실패의 **질**이 달랐다.
이전의 실패: "rank()로 감싸기" 수준의 단순 반복.
새로운 실패: "BRAIN 문법 확인 안 함"이라는 구체적이고 해결 가능한 문제.

이 차이가 ICQ3 이후의 진전을 가능하게 했다.

## 사이클 감지의 실전 교훈

사이클 위치 감지는 이론적으로 가장 강력했다.
IC가 5배 차이 나는 조건을 알면, 조건부 전략만으로도 Sharpe가 두 배가 된다.

하지만 실전에서 배운 것:

1. **bucket()보다 곱셈 필터가 안정적이다.**
   이산적 분류의 경계 불안정이 TVR을 죽인다.

2. **시장 레벨 사이클을 종목 레벨로 번역하면 정보가 손실된다.**
   종목 자체 vol 비율은 시장 전체 사이클의 약한 프록시다.

3. **조건부 전략은 "무조건부 + 필터"보다 낫지만, 필터의 정확도가 관건이다.**
   잘못된 필터는 좋은 신호를 죽인다.
