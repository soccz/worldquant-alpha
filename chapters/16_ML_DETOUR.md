# 16장: 딥러닝 우회로 — 복잡함이 배신한 순간

---

## 수개월을 딥러닝에 쏟았다. 결과는 충격적이었다.

알파 수식을 손으로 작성하는 것에 한계를 느끼고 있었다.
460개를 만들어서 2%만 통과했다.
인간의 직관으로는 한계가 있다고 생각했다.

딥러닝이 답일 수 있다고 믿었다.
시장 상태(변동성 체제, 지수 모멘텀, 추세 위치)를 Transformer에 주입하면,
모델이 비선형 상호작용을 알아서 발견할 것이라고.

연구 질문은 단순했다:
**시장 상태 정보를 어떻게 모델에 주입하는가?**

5가지 방법을 비교했다.
결과는 내 가정을 완전히 뒤엎었다.

## 5가지 컨디셔닝 인터페이스

동일한 기본 아키텍처(Transformer 2층 4헤드 + TCN 3층)에
동일한 시장 상태 정보(position, intensity, indexret)를
5가지 다른 방식으로 주입했다.

| # | 인터페이스 | 유형 | 핵심 메커니즘 |
|---|----------|------|------------|
| 1 | **concat_a** | 덧셈적 | 시장 상태를 입력에 붙임. 가장 단순 |
| 2 | econ_time (PE) | 덧셈적 (지연) | 시장 상태로 positional encoding 변조 |
| 3 | tau-RoPE | 곱셈적 (회전) | "경제적 시간"으로 attention 각도 변형 |
| 4 | FiLM | 곱셈적 | gamma(c) * h + beta(c)로 은닉층 변조 |
| 5 | XIP | 덧셈적 (명시적) | intensity x indexret 상호작용을 명시 계산 |

모든 인터페이스의 파라미터 수를 1% 이내로 통일했다.
옵티마이저, 학습률, 배치 크기, 에폭 수 모두 동일.
시드 3개(7, 17, 27)로 안정성 확인.

공정한 비교였다. 유일한 변수는 시장 상태의 "주입 방식"이었다.

## "코드 버그인 줄 알았다"

FiLM의 결과를 처음 봤을 때, 당연히 코드 버그라고 생각했다.

곱셈적 인터페이스가 덧셈적 인터페이스보다 **이론적으로 우월**하다는 건
ML 커뮤니티의 상식이었다.

- Jayakumar et al. (2020, ICLR): 곱셈적 상호작용이 이중선형 함수 클래스를
  효율적으로 표현함을 수학적으로 증명
- Perez et al. (2018): FiLM이 시각적 추론(CLEVR)에서 명확히 우세
- Peebles & Xie (2023): DiT에서 adaLN-Zero(곱셈적)가 입력 결합을 명확히 이김

이 모든 문헌이 "곱셈적을 써라"라고 말하고 있었다.

그래서 3번 재검증했다:

1. **코드 리뷰**: FiLM 구현을 한 줄씩 확인. gamma와 beta가 올바르게 적용되는지.
   버그 없었다.

2. **하이퍼파라미터 스윕**: 학습률을 1e-2부터 1e-5까지. 정규화를 바꾸고,
   초기화를 바꾸고, 드롭아웃을 넣고 빼고. 안정화되지 않았다.

3. **시드 분석**: 3개 시드에서 FiLM의 IC 범위: -0.079 ~ +0.076.
   시드 7에서는 양수, 시드 17에서는 극단적 음수.
   같은 코드, 같은 데이터, 같은 설정인데 시드만 바꾸면 결과가 반전.

버그가 아니었다. 진짜로 FiLM이 실패한 것이었다.

## 왜: 저 SNR에서 곱셈적 잡음 전파

금융 데이터의 신호 대 잡음비(SNR)는 약 0.2다.
이것이 모든 것을 설명했다.

### 곱셈적 경로 (FiLM)에서의 잡음 전파

```
h_FiLM = gamma(c + epsilon) * f(x) + beta(c + epsilon)
       = gamma(c) * f(x) + beta(c) + epsilon * [gamma'(c) * f(x) + beta'(c)]
```

잡음 epsilon이 **f(x) 전체에 곱해져** 전파된다.
은닉 표현의 norm이 크면, 잡음의 영향이 증폭된다.

### 덧셈적 경로 (concat)에서의 잡음 전파

```
h_concat = g(W @ [x; c + epsilon] + b)
         = g(W_x @ x + W_c @ (c + epsilon) + b)
```

잡음 epsilon은 선형 투사 W_c를 통과한 후 비선형 함수 g에 의해 감쇠된다.
ReLU, tanh 등의 활성화 함수는 작은 입력 변동을 흡수한다.

### 핵심 차이

곱셈적 경로: 잡음이 은닉 표현 전체를 변조
덧셈적 경로: 잡음이 입력 공간의 한 차원에 국한

FiLM 학습 동역학을 추적한 결과:

| 시드 | gamma mean (final) | gamma std (final) | IC |
|------|-------------------|-------------------|-----|
| 7 | 1.003 | 0.012 | +0.076 |
| 17 | 0.997 | 0.089 | -0.079 |
| 27 | 1.001 | 0.031 | -0.017 |

시드 7: gamma가 1 근처에 안정 → 사실상 항등 변환 → concat에 가까운 행동 → IC 양수.
시드 17: gamma의 표준편차가 폭발(0.089) → 잡음 증폭 → IC 대폭 음수.

FiLM이 "잘 작동할 때"는 사실 **FiLM이 아무 것도 안 할 때**였다.
gamma = 1, beta = 0이면 FiLM은 항등 변환이다. 없는 것과 같다.

## 채널 분해: 핵심 신호의 정체

왜 concat이 이겼는지를 더 깊이 파기 위해 채널 분해를 했다.

| 채널 조합 | IC |
|----------|-----|
| intensity + position + indexret (전체) | 0.0571 |
| intensity + indexret | 0.0592 |
| intensity only | 0.0066 |
| position only | 0.0188 |
| indexret only | 0.0205 |
| 선형 앙상블 (개별 합) | ~0.019 |

**상호작용 효과: 결합 IC(0.059) - 선형 합(0.019) = 0.040.**
전체 신호의 ~68%가 채널 간 비선형 상호작용에서 왔다.

intensity 단독 IC = 0.007. 거의 무의미하다.
indexret 단독 IC = 0.021. 약하다.
하지만 둘을 함께 주면 IC = 0.059. **2배 이상.**

이 상호작용(intensity x indexret)이 핵심 예측 신호였다.
"고변동성(높은 intensity) + 강한 지수 모멘텀(높은 indexret)"이
결합될 때만 교차 단면 수익률 예측력이 급증하는 패턴.

증분적 식별 검정: F = 14.335, p = 6.09e-07.
intensity x indexret 상호작용이 통계적으로 매우 유의미했다.

### concat이 이 상호작용을 발견하는 메커니즘

```
[x; intensity; indexret] → W_proj → 각 뉴런이 intensity와 indexret에 동시 접근
```

첫 번째 투사층에서 각 뉴런이 intensity와 indexret에 동시에 접근하여
비선형 상호작용을 스스로 발견할 수 있다.

### FiLM이 이 상호작용을 발견할 수 없는 이유

```
FiLM: gamma(c) * f(x) + beta(c)
```

gamma와 beta는 시장 상태 c에서 계산되지만,
f(x)는 이미 자산 피처만으로 계산된 은닉 표현이다.
gamma가 f(x)를 스케일링하는 방식으로는
intensity와 indexret의 상호작용을 자산 피처 공간에 주입할 수 없다.

## tau-RoPE: 이론적으로 가장 아름다웠다

Clark(1973)의 subordinated process 이론:
자산 가격은 달력 시간이 아닌 "경제적 시간"에 따라 움직인다.
시장 활동이 높을 때 시간이 "빨리 흐른다."

이것을 Transformer의 RoPE(회전 위치 인코딩)에 적용했다.
attention의 시간 거리를 "경제적 시간"으로 계산하면,
고변동성 기간에는 시간이 압축되고 저변동성 기간에는 팽창된다.

이론적으로 가장 아름다운 아이디어였다.

### 결과

MAE(평균 절대 오차)는 개선됐다. p = 0.0019. 통계적으로 유의미.

하지만 IC(순위 상관, 즉 알파 예측력)는 변화 없었다. p = 0.0595.

"시간 좌표를 바꾸면 더 잘 맞추지만, 순서를 더 잘 매기지는 못한다."

### 3가지 병목

**병목 1: Softmax 압축.**
QK dot product에서의 미세한 순서 변화(delta ~ 1e-05)가
softmax를 통과하면서 사라진다.
softmax의 온도(sqrt(d_k))보다 작은 변화는 출력에 거의 영향 없다.

**병목 2: 상호작용 접근 제약.**
tau-RoPE 경로: intensity → step → tau → RoPE rotation angle.
이 경로에서 indexret은 참여하지 않는다.
핵심 상호작용(intensity x indexret)을 구조적으로 포착할 수 없다.

**병목 3: 단조성 제약.**
tau = cumsum(step)으로 구성되므로 반드시 단조증가.
물리적 시간과의 상관이 0.999996. 사실상 동일하다.
"경제적 시간"이라고 했지만 달력 시간과 구분이 안 된다.

## Paper C: 3가지 근본 병목

실패 분석 논문(Paper C)에서 10개 가설을 체계적으로 검정했다.

| 가설 | 결과 | 함의 |
|------|------|------|
| H1: learned_tau > concat | **실패** | 성능 우위 없음 |
| H2: geometry 변화 → 예측 개선 | **실패** | 표현 변화 =/= 예측 유용성 |
| H3: alignment 달성 가능 | 성공 | 학습은 가능하나 불충분 |
| H5: window signature 충분 | **실패** | IC = -0.0226, 기준선 이하 |
| H8: 순서 보존 중요 | **실패** | shuffled > ordered |
| H9: 채널 상호작용 핵심 | **핵심 확인** | 상호작용이 신호의 68% |
| H10: FiLM 안정적 대안 | **실패** | 극도로 불안정 (-0.08 ~ +0.08) |

가장 고통스러운 결과:
H8에서 입력을 셔플해도(시간 순서를 파괴해도) 성능이 나빠지지 않았다.
오히려 약간 좋아졌다 (IC: 0.0187 vs 0.0066).

시간 순서가 중요하지 않다는 것은
Transformer가 시계열의 "순서"가 아니라 "분포"를 학습하고 있다는 뜻이다.

## 고 SNR 제어 실험: FiLM의 역전

"FiLM이 SNR이 높으면 이기는가?"를 직접 검증했다.

컨디셔닝 신호를 이산화(discretization)하여 SNR을 인위적으로 높였다.
연속형 intensity를 3-bucket 범주형으로 변환하면 잡음이 크게 줄어든다.

결과: **FiLM이 concat을 역전.** delta = +0.047.

SNR 이론의 양방향 예측이 실제 금융 데이터에서 확인됐다:
- 저 SNR (연속형 시장 지표) → concat 우세
- 고 SNR (이산화된 범주형) → FiLM 우세

교차점 SNR* = 0.2 근방.
금융 데이터의 유효 SNR은 이보다 낮다. 그래서 concat이 이긴다.

## 알파 생성에의 교훈: 놀라운 평행 관계

이 딥러닝 실험의 결과는 알파 수식 작성에 직접적 교훈을 준다.

### rank(A*B) vs rank(A)*rank(B)

알파 수식에서도 같은 문제가 있다:

```
Interface 2 (Early Mult): rank(A * B)
→ 두 신호가 같은 공간에서 먼저 결합 → 상호작용 포착

Interface 1 (Late Mult): rank(A) * rank(B)
→ 각각 독립적으로 정규화 후 결합 → 상호작용 손실 가능
```

딥러닝에서 concat(조기 결합)이 FiLM(지연 결합)을 이긴 것처럼,
알파에서도 rank(A*B)(조기 결합)가
rank(A)*rank(B)(지연 결합)을 이길 수 있다.

27_SIGNAL_INTERFACE_THEORY의 인터페이스 분류와 정확히 일치하는 발견이다.

### 복잡함은 약한 신호에서 도움이 안 된다

금융 데이터의 SNR = 0.2 수준에서:
- 이론적으로 우아한 FiLM → 실패
- 이론적으로 아름다운 tau-RoPE → 무의미한 개선
- 가장 단순한 concat → 승리

알파 수식에서도:
- 4중 구조(느린 x 빠른 x 미반영 x 체제) → 신호 희소, 과적합
- 단순한 2~3 요소 조합 → robust, 안정적 Sharpe

**Complexity doesn't help when signal is weak. Robustness > sophistication.**

이것이 460개 알파의 실패와 딥러닝 실험의 실패를 관통하는 단일 교훈이다.

## 교차 시장 복제의 부정적 결과

GSPC(S&P 500) 앵커에서 발견한 intensity x indexret 상호작용이
IXIC(Nasdaq) 앵커에서도 재현되는지 검증했다.

결과: IC = 0.020 (S&P 500의 0.059 대비 약 1/3).

핵심 상호작용 패턴이 시장 의존적이었다.
하나의 시장에서 발견한 패턴이 다른 시장에서 약화된다는 것은
일반화에 제한이 있다는 정직한 인정이다.

이것은 알파에서도 마찬가지다:
TOP3000에서 통과한 알파가 TOP200에서 실패하는 이유.
데이터에서 발견한 패턴은 항상 해당 데이터에 과적합 위험이 있다.

## 잡음 주입 절제: 예상치 못한 발견

FiLM에 의도적으로 가우시안 잡음을 주입하는 실험을 했다.
적정 수준의 잡음(noise = 1.0)이 FiLM의 성능을 **개선**했다.

비구조적 잡음이 FiLM에 정규화 효과를 준 것이다.
잡음이 gamma의 과적합을 방지하고 일반화를 도왔다.

이것도 알파에 교훈이 있다:
Decay를 높이는 것(일종의 시간적 잡음 주입)이
알파의 과적합을 방지하고 Fitness를 개선하는 것과 같은 원리다.

## 수개월의 비용

딥러닝 실험에 수개월을 투자했다.
5가지 인터페이스를 설계하고, 수백 번의 학습을 돌리고,
3편의 논문 초고를 작성했다.

결론은 이것이었다:
"가장 단순한 방법이 가장 좋다."

이 결론을 얻기 위해 수개월이 필요했다는 것이
딥러닝 우회로의 진짜 비용이다.

하지만 이 우회로가 없었다면:
- 신호 인터페이스 이론(6가지 결합 방법)을 발견하지 못했을 것
- "SNR이 낮을 때 복잡함이 해롭다"는 원칙을 체화하지 못했을 것
- 조기 결합 vs 지연 결합의 차이를 이해하지 못했을 것

실패한 실험이 가장 깊은 통찰을 준 사례다.

## 핵심 정리

1. **가장 단순한 방법(concat)이 가장 우아한 방법(FiLM, tau-RoPE)을 이겼다.**
   IC: 0.057 vs -0.008. 차이가 0.065 — 전체 신호보다 크다.

2. **왜: 금융 데이터의 SNR이 너무 낮다 (약 0.2).**
   곱셈적 인터페이스는 잡음을 증폭한다. 덧셈적 인터페이스는 잡음을 흡수한다.

3. **핵심 신호의 68%가 채널 간 상호작용(intensity x indexret)에 있었다.**
   개별 채널은 거의 무의미. 결합이 전부.

4. **tau-RoPE는 MAE를 개선했지만(p=0.0019) IC는 변화 없었다(p=0.0595).**
   "더 잘 맞추는 것"과 "순서를 더 잘 매기는 것"은 다르다.

5. **이 교훈은 알파 수식에도 적용된다.**
   rank(A*B) vs rank(A)*rank(B) — 같은 lesson, different world.
   Complexity doesn't help when signal is weak.
